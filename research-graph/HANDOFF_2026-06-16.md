### 2. Run the cheapest CF-1-threatening tests first (all ~30 min local CPU)
The 4 briefs propose attacks on CF-1 (free length+logprob gate = most generalizing
zero-cost readout). Ranked by value-per-cost:

1. **CH-57 / SmolLM2 — AUROC vs base capability** *(HIGH, ~30min CPU, pure regression)*
   Regress pinned free-gate OOF AUROCs (SmolLM2 0.810 / Gemma 0.844 / OLMo-2 0.838 +
   a strong-math base) on each base's GSM8K/MATH accuracy. If AUROC trends *down* as
   base accuracy rises, the 0.810 anchor is partly inflated by base mediocrity and
   CF-8's base-swap (→ Qwen2.5-Math) erodes the gate. **Questions the headline anchor;
   needs no new generations.** Start here.
2. **CH-57 / NAT — contrastive prefix logprob** *(MED, ~30min CPU, no training)*
   Re-score *cached* generations under "generate a correct solution" vs "…incorrect…"
   prefixes; OOF logistic on Δ = logprob(correct) − logprob(incorrect) vs free
   baseline. A cheap readout that *could* beat the free gate.
3. **CH-20 — lexical TF-IDF preflight** *(HIGH, ~30min CPU)*
   TF-IDF 1–3-gram logreg on MATH/BBH prompt strings vs pinned length-only 0.7056.
   CF-7 only tested length + geometric eigenspectrum, never lexical content — a real
   gap. **Snag:** raw prompt strings must be recovered from the gate cache (not in
   `pinned_meta.json`).
4. **CH-58 / SmolLM2 — length-control decoupling** *(MED, ~30min CPU)*
   length-only AUROC on base vs Instruct SmolLM2-1.7B. If Instruct's length signal is
   weaker, "length carries the gate" is a post-training artifact, not universal.
